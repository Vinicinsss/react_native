# Glossário de domínio

## 1. Termos do negócio
- **Objetivo / Meta:** Item que precisa ser cumprido no dia. Sinônimos proibidos: "task", "tarefa" (para manter o foco na palavra objetivo). Exemplo de uso: "Criar um novo objetivo de leitura".
- **Ciclo Diário:** Período de 24 horas onde os objetivos ficam ativos. Termina sempre à meia-noite com a limpeza da lista.
- **Checklist:** A lista visual na interface onde é possível marcar os objetivos como concluídos.

## 2. Termos técnicos
- **React Native:** Framework utilizado para a criação do aplicativo mobile.
- **Clean Architecture:** Padrão arquitetural utilizado no back-end para separar as regras de negócio das camadas de infraestrutura.
- **SQL Server:** Sistema de banco de dados onde os objetivos serão persistidos.

## 3. Convenções de nomenclatura
- Tabelas do banco de dados devem ser nomeadas no plural (ex: `Objetivos`).
- Endpoints da API devem ser escritos em minúsculo e separados por hífen (ex: `/api/v1/objetivos-diarios`).
- Variáveis no código devem utilizar o padrão camelCase (ex: `objetivoId`).
